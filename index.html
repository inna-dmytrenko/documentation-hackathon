Hackathon Documentation: Testing Vacation Booking Site

Project Description

During the hackathon, we tested a vacation booking site. This site allows users to search for, view, and book hotel rooms for selected dates.

Testing Objectives

The main objectives of the testing were to verify the site's functionality, usability, security, and performance. Specifically, we focused on the following aspects:

  • Correctness of room search
  • Accuracy of room information display
  • Booking process
  • Payment processing
  • User data security
  • Compatibility with different browsers and devices

Test Scenarios

Test Scenario 1: Room Search

Description: Verify the functionality of searching for rooms with specific parameters.

Steps:

  1. Open the main page of the site.
  2. Enter the destination in the search field.
  3. Select check-in and check-out dates.
  4. Specify the number of guests.
  5. Click the "Search" button.

Expected Result: A list of available rooms matching the specified parameters is displayed.

Test Scenario 2: Viewing Room Information

Description: Verify the display of detailed room information.

Steps:

  1. Select any room from the search results.
  2. View the room information, including description, photos, prices, and reviews.

Expected Result: All information is displayed correctly, images load, and reviews are accessible for viewing.

Test Scenario 3: Booking Process

Description: Verify the booking process for a room.

Steps:

  1. Select a room from the search results.
  2. Click the "Book" button.
  3. Fill in the required information (name, surname, contact details).
  4. Select a payment method and enter payment details.
  5. Click the "Confirm Booking" button.

Expected Result: The user receives a booking confirmation with relevant information displayed on the screen and sent to their email.

Test Scenario 4: Payment Processing

Description: Verify the correctness of payment processing.

Steps:

  1. Proceed with the booking process until the payment details step.
  2. Enter payment details (e.g., credit card number).
  3. Confirm the payment.

Expected Result: Payment is processed without errors, and the user receives a successful transaction confirmation.

Test Scenario 5: Data Security

Description: Verify the security of user data.

Steps:

  1. Check if secure protocols (HTTPS) are used on all site pages.
  2. Conduct tests for SQL injection and other common web vulnerabilities.
  3. Ensure personal and payment data are stored and transmitted securely.

Expected Result: The site is protected against major types of attacks, and user data is transmitted and stored securely.

Test Scenario 6: Browser and Device Compatibility

Description: Verify the correct display of the site on different browsers and devices.

Steps:

  1. Open the site on different browsers (Chrome, Firefox, Safari, Edge).
  2. Check the correct display and functionality on various devices (desktop computers, laptops, tablets, smartphones).

Expected Result: The site displays and functions correctly on all tested browsers and devices.

Testing Results

  • Room Search: Test passed successfully. All tested search parameters work correctly.
  • Viewing Room Information: All information is displayed correctly, and no errors were found.
  • Booking Process: The booking process works smoothly, and the user receives confirmation.
  • Payment Processing: Payments are processed correctly, and transaction confirmations are received.
  • Data Security: Several vulnerabilities were identified and promptly fixed. The site now meets security standards.
  • Compatibility: The site works correctly on all tested browsers and devices.

Recommendations

  • Regularly conduct security checks to identify and fix new vulnerabilities.
  • Improve responsive design for a better user experience on mobile devices.
  • Implement automated testing to ensure site stability after updates.

Conclusion

Testing during the hackathon identified a few minor issues that were successfully resolved. Overall, the site operates stably and securely, providing users with a convenient way to search for, view, and book vacation rooms.
